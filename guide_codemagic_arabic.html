1. نظرة عامة: ما هو Codemagic؟

Codemagic هو نظام بناء سحابي (CI/CD) مصمم خصيصاً لتطبيقات الموبايل. يعني بدل ما تبني الـ APK على جهازك، الكود يتروش لسيرفرات Codemagic في السحابة، وهني يتم بناء الـ APK تلقائياً، وبعدين تقدر تنزل الملف الجاهز على تابلتك مباشرة من المتصفح.

هالخدمة كانت تركّز على Flutter في البداية، بس حالياً تدعم Android Native (Kotlin/Java) بشكل كامل، وكذلك iOS وReact Native. يعني مشروع Breathy اللي مكتوب بـ Kotlin + Jetpack Compose مدعوم 100%.

لماذا Codemagic هو الخيار الأفضل لك؟

أنت تستخدم تابلت Android بدون كمبيوتر. هالوضع ما يمنعك من بناء تطبيقك! Codemagic يحل المشكلة بالكامل لأنه يشتغل من المتصفح فقط. ما تحتاج تثبت أي برنامج على التابلت، ما تحتاج JDK أو Gradle أو Android SDK. كل اللي تحتاجه هو متصفح إنترنت وبريد إلكتروني.

العملية ببساطة: ترفع الكود إلى GitHub (من المتصفح)، وتخلي Codemagic يبني لك الـ APK في السحابة، وبعدين تنزل الملف النهائي وتثبته على تابلتك. كل خطوة تتم من متصفح التابلت بدون أي أداة إضافية.

معلومة مهمة

عندنا ملف إعدادات جاهز codemagic.yaml مضاف لمشروع Breathy. هالملف يخلي Codemagic يعرف إشلون يبني التطبيق بالضبط بدون أي إعداد يدوي معقد.

2. مقارنة طرق البناء السحابية

في عدة طرق تقدر تبني الـ APK من التابلت بدون كمبيوتر. الجدول التالي يقارن أهم الطرق المتاحة عشان تختار الأنسب لك:

الطريقة مجاني؟ يدعم Android Native سهولة من التابلت سرعة البناء
Codemagic نعم (خطة مجانية) مدعوم بالكامل سهل جداً - من المتصفح 5-15 دقيقة
GitHub Actions نعم (2000 دقيقة/شهر) مدعوم بالكامل سهل - من المتصفح 5-15 دقيقة
Termux مجاني مدعوم صعب - يحتاج إعداد كثير بطيء (30-60 دقيقة)
AIDE IDE مجاني محدود Java فقط (لا Kotlin) متوسط متوسط
Firebase App Distribution مجاني يحتاج بناء من مكان ثاني لا يبني - يوزع فقط -
توصيتنا

ننصح بـ Codemagic كخيار أول لأنه مصمم خصيصاً لتطبيقات الموبايل، وواجهته سهلة جداً من المتصفح. GitHub Actions هو بديل ممتاز أيضاً وعندنا ملف workflow جاهز له. الطريقتين يشتغلون من متصفح التابلت بدون أي مشكلة.

ملاحظة عن Termux

Termux يخليك تثبت JDK وGradle على التابلت وتبني الـ APK محلياً. بس العملية معقدة وتحتاج أوامر كثيرة في التيرمنال، والبناء يكون بطيء لأن التابلت أضعف من سيرفرات السحابة. مع ذلك، هو خيار مجاني بالكامل ويعمل بدون إنترنت بعد التثبيت. لو حاب تجربه، في قسم منفصل نشرح الخطوات.

3. المتطلبات قبل البدء

قبل ما تبدأ عملية البناء، تأكد من توفر الأشياء التالية:

متطلبات أساسية

  • تابلت Android - أي تابلت يشغل Chrome أو أي متصفح حديث
  • اتصال إنترنت - مستقر ويفضل Wi-Fi لأن ملفات البناء كبيرة شوي
  • بريد إلكتروني - لتسجيل حساب GitHub وحساب Codemagic
  • ملفات مشروع Breathy - اللي عندك جاهزة (51+ ملف Kotlin + إعدادات)

ما تحتاجه (لا يلزم!)

  • لا تحتاج كمبيوتر أو لابتوب
  • لا تحتاج تثبت Android Studio أو JDK
  • لا تحتاج تثبت أي تطبيق على التابلت
  • لا تحتاج خبرة في سطر الأوامر

الوقت المتوقع

العملية كاملة من البداية (إنشاء حساب) لحد ما تنزل الـ APK على التابلت تاخذ تقريباً 30-45 دقيقة في أول مرة. المرات الجاية تكون أسرع لأن الحسابات جاهزة.

تنبيه مهم

البناء السحابي يحتاج إنترنت مستقر. لو الانقطع أثناء رفع الكود أو تحميل الـ APK، أعد المحاولة. البيانات ما تضيع لأن كل شي محفوظ في GitHub وCodemagic.

4. الخطوة 1: إنشاء حساب GitHub

GitHub هو المكان اللي تحفظ فيه كود المشروع. Codemagic يقرأ الكود من GitHub ويبني الـ APK.

1 افتح متصفح Chrome على التابلت

اذهب إلى العنوان: https://github.com/signup

2 أدخل بيانات التسجيل

اكتب بريدك الإلكتروني، واختر اسم مستخدم (username)، وكلمة مرور قوية. اضغط "Continue" بعد كل خطوة.

3 تحقق من البريد الإلكتروني

GitHub يرسل كود تحقق لبريدك. انسخ الكود وأدخله في الصفحة.

4 أكمل الإعدادات

اختر الخطة المجانية (Free). مو لازم تدخل بيانات الدفع.

لو عندك حساب GitHub أصلاً

سجّل دخولك على https://github.com/login وانتقل للخطوة التالية مباشرة.

5. الخطوة 2: رفع كود المشروع إلى GitHub

الحين لازم ترفع كل ملفات مشروع Breathy إلى GitHub. عندك طريقتين:

الطريقة أ: رفع من المتصفح (الأسهل)

1 أنشئ مستودع جديد

اذهب إلى https://github.com/new - اكتب اسم المستودع: breathy - اختر Public - اضغط "Create repository"

2 ارفع ملفات المشروع

في صفحة المستودع الفارغ، اضغط "uploading an existing file" - اسحب وأفلت كل ملفات المشروع أو اخترها من التابلت - اضغط "Commit changes"

ملاحظة عن رفع الملفات

واجهة GitHub ما تسمح برفع مجلدات مباشرة. لازم تنشئ المجلدات واحدة واحدة باستخدام "Add file" ثم "Create new file" واكتب المسار مثل: app/src/main/java/com/breathy/MainActivity.kt - هالطريقة تنشئ المجلدات تلقائياً. البديل هو استخدام تطبيق GitHub المجاني من Play Store اللي يخليك ترفع مجلدات كاملة.

الطريقة ب: تطبيق GitHub من Play Store (الأسهل للمجلدات)

1 ثبّت تطبيق GitHub

افتح Google Play وابحث عن "GitHub" وثبّت التطبيق الرسمي.

2 سجّل دخولك وأنشئ مستودع

افتح التطبيق > سجّل دخولك > اضغط "+" > أنشئ مستودع جديد اسمه "breathy"

3 ارفع الملفات

استخدم مدير الملفات لنسخ مجلد مشروع Breathy كامل إلى المستودع. التطبيق يدعم رفع مجلدات كاملة بسهولة.

هيكلة الملفات المطلوبة

تأكد إن الملفات مرفوعة بالهيكلة الصحيحة. أهم الملفات اللي لازم تتأكد منها:

  • build.gradle.kts - ملف البناء الرئيسي (في الجذر)
  • app/build.gradle.kts - ملف بناء التطبيق
  • settings.gradle.kts - إعدادات Gradle
  • gradle/libs.versions.toml - إصدارات المكتبات
  • gradle.properties - خصائص Gradle
  • app/google-services.json - إعدادات Firebase
  • codemagic.yaml - إعدادات Codemagic
  • app/src/main/java/com/breathy/*.kt - كل ملفات Kotlin

6. الخطوة 3: إنشاء حساب Codemagic

1 اذهب لموقع Codemagic

افتح المتصفح واذهب إلى: https://codemagic.io/signup

2 سجّل باستخدام GitHub

اضغط على زر "Sign up with GitHub" - هالخيار أسهل لأنه يربط حسابك مباشرة بدون إعداد إضافي. Codemagic يطلب صلاحية قراءة المستودعات من GitHub، وافق على ذلك.

3 أكمل التسجيل

أدخل معلوماتك الأساسية (اسمك، نوع المشروع Android) - اختر الخطة المجانية Free - لا تحتاج تدخل بيانات دفع.

الخطة المجانية

خطة Codemagic المجانية تعطيك 500 دقيقة بناء شهرياً على Linux. بناء مشروع Breathy ياخذ تقريباً 5-15 دقيقة، يعني تقدر تبني حوالي 30-50 مرة بالشهر مجاناً. أكثر من كافي للتطوير والتجربة.

7. الخطوة 4: ربط المشروع وإعدادات البناء

1 أضف التطبيق

بعد تسجيل الدخول، اضغط "Add application" - اختر GitHub كمصدر - اختر مستودع breathy من القائمة - اختر نوع المشروع "Android App" - اضغط "Finish"

2 اختر إعدادات البناء

Codemagic يكشف ملف codemagic.yaml تلقائياً من المستودع. هالملف يحتوي كل الإعدادات الجاهزة. بس تأكد من الآتي:

  • Build format: Android App (APK)
  • Build mode: Debug (للتجربة) أو Release (للنشر)
  • Android module: app
3 إعداد المتغيرات البيئية (Environment Variables)

في صفحة إعدادات البناء، اذهب لقسم "Environment variables" وأضف المتغيرات التالية:

اسم المتغير القيمة مطلوب؟
GOOGLE_SERVICES_JSON محتوى ملف google-services.json نعم - ضروري
DEBUG_KEYSTORE_PASSWORD android للبناء Debug
DEBUG_KEY_ALIAS androiddebugkey للبناء Debug
DEBUG_KEY_PASSWORD android للبناء Debug
كيف تضيف محتوى google-services.json

افتح الملف على التابلت > انسخ كل المحتوى > في Codemagic أضف متغير جديد اسمه GOOGLE_SERVICES_JSON > الصق المحتوى > اختر "Secure" (مشفّر) > احفظ. الـ script في codemagic.yaml بيكتب هالمحتوى في الملف الصحيح أثناء البناء تلقائياً.

8. الخطوة 5: بناء الـ APK وتحميله

1 ابدأ البناء

في صفحة المشروع على Codemagic، اضغط زر "Start new build" - اختر الـ workflow المناسب: android-debug (للتجربة) - اضغط "Start build".

2 راقب تقدم البناء

صفحة البناء تعرض سجل مباشر (live log) لكل خطوة. تقدر تتابع التقدم بالتفصيل. البناء يمر بالمراحل التالية:

  • تحميل الكود من GitHub (1-2 دقيقة)
  • إعداد البيئة - JDK + Android SDK (2-3 دقائق)
  • تحميل المكتبات - Gradle sync (3-5 دقائق)
  • بناء الـ APK - compile + package (5-8 دقائق)
  • رفع الـ artifacts - الملف النهائي (1 دقيقة)
3 حمّل الـ APK

بعد ما ينتهي البناء بنجاح (علامة خضراء)، اضغط على تبويب "Artifacts" - حتلاقي ملف الـ APK جاهز للتحميل - اضغط على اسم الملف وسيبدأ التحميل على تابلتك.

البريد الإلكتروني

Codemagic يرسل لك إشعار على البريد الإلكتروني بعد انتهاء البناء (نجاح أو فشل) مع رابط مباشر لتحميل الـ APK. تقدر تحمّله من الإيميل بدل ما تدخل الموقع.

في حال فشل البناء

لو البناء فشل، لا تقلق! اضغط على البناء الفاشل وشوف الـ log. أكثر أسباب الفشل شيوعاً:

  • ملف google-services.json مفقود أو خطأ - تأكد إن المتغير البيئي يحتوي المحتوى الصحيح
  • خطأ في ملف Kotlin - شوف رقم السطر في الـ log واصلح الكود في GitHub
  • مشكلة في Gradle sync - غالباً بسبب إصدارات المكتبات، شوف الـ log وعدّل الملف المطلوب

بعد إصلاح المشكلة، ارفع الكود المعدّل على GitHub وابدأ بناء جديد. Codemagic يبني النسخة الأحدث تلقائياً.

9. الخطوة 6: تثبيت الـ APK على التابلت

1 فعّل تثبيت من مصادر غير معروفة

اذهب إلى: الإعدادات > الأمان والخصوصية > تثبيت تطبيقات غير معروفة > فعّل الخيار لمتصفح Chrome أو مدير الملفات. على بعض الأجهزة: الإعدادات > التطبيقات > وصول خاص > تثبيت تطبيقات غير معروفة.

2 افتح ملف الـ APK

بعد تحميل الـ APK، اذهب إلى مجلد التنزيلات (Downloads) على التابلت - اضغط على ملف الـ APK (اسمه مثل: app-debug.apk) - اضغط "تثبيت".

3 افتح التطبيق

بعد التثبيت، اضغط "فتح" أو حتلاقي أيقونة Breathy في قائمة التطبيقات. التطبيق حيطلب منك تسجيل الدخول أو إنشاء حساب. استمتع!

تنبيه عن توقيع التطبيق

نسخة Debug موقّعة بمفتاح تجريبي (debug keystore). هالنسخة ممتازة للتجربة والاختبار على جهازك. بس لنشر التطبيق على Play Store لازم تبني نسخة Release بمفتاح توقيع حقيقي (release keystore). هذا موضوع منفصل لمرحلة النشر.

10. حل المشاكل الشائعة

مشكلة: البناء يفشل مع خطأ "google-services.json not found"

هذي أكثر مشكلة شيوعاً. الحل: تأكد إنك أضفت المتغير البيئي GOOGLE_SERVICES_JSON في إعدادات Codemagic، وتأكد إن المحتوى منسوخ بالكامل بدون حذف أي حرف. الملف لازم يبدأ بـ { وينتهي بـ }. لو الـ script ما يشتغل، جرب ترفع الملف مباشرة في المستودع بدل المتغير البيئي.

مشكلة: خطأ في تجميع Kotlin "Unresolved reference"

هالخطأ يعني إن في ملف Kotlin فيه اسم كلاس أو دالة غلط. شوف الـ log وروح لرقم السطر المذكور، وصلّح الكود في GitHub. بعد التعديل، ابدأ بناء جديد.

مشكلة: البناء ياخذ وقت طويل جداً (أكثر من 30 دقيقة)

أول بناء يكون أبطأ لأن Codemagic يحمل كل المكتبات لأول مرة. المرات الجاية تكون أسرع بكثير بسبب الـ cache. لو ياخذ أكثر من 30 دقيقة، تأكد إن ملف gradle.properties موجود وفيه إعدادات الـ caching.

مشكلة: "Out of memory" أثناء البناء

لو السيرفر يوصل الذاكرة القصوى، جرب تبني Debug بدل Release (لأن Release يشغل Proguard ويستهلك ذاكرة أكثر). أو غيّر نوع السيرفر من Linux x1 إلى Linux x2 في إعدادات البناء.

مشكلة: الـ APK ما يثبت على التابلت

تأكد من تفعيل "تثبيت من مصادر غير معروفة" في الإعدادات. لو تطلع رسالة "App not installed"، ممكن يكون في نسخة سابقة من التطبيق مثبتة بنفس اسم الحزمة. احذف النسخة القديمة أولاً ثم ثبّت الجديدة.

مشكلة: التطبيق يفتح بس يوقف (crash)

ممكن يكون في مشكلة في الاتصال بـ Firebase. تأكد إن google-services.json يحتوي البيانات الصحيحة لمشروعك. وشوف الـ logcat (لو تقدر) لمعرفة سبب التوقف بالتفصيل.

11. الطريقة البديلة: GitHub Actions

لو ما تبي تستخدم Codemagic، في طريقة ثانية مجانية بالكامل عبر GitHub Actions. عندنا ملف workflow جاهز مضاف للمشروع في المسار:

.github/workflows/build-apk.yml

كيف تستخدم GitHub Actions

1 ارفع الكود إلى GitHub

نفس الخطوات المذكورة قبل - تأكد إن ملف .github/workflows/build-apk.yml مرفوع مع المشروع.

2 أضف الـ Secrets

اذهب إلى صفحة المستودع > Settings > Secrets and variables > Actions > أضف الـ secrets التالية:

  • GOOGLE_SERVICES_JSON - محتوى ملف Firebase
3 شغّل البناء يدوياً

اذهب إلى تبويب "Actions" في المستودع > اختر "Build Breathy APK" > اضغط "Run workflow" > اختر فرع main > اضغط "Run workflow".

4 حمّل الـ APK

بعد انتهاء البناء (5-15 دقيقة)، اضغط على الـ workflow run > اسحب لأسفل لقسم "Artifacts" > حمّل breathy-debug-apk. الملف حيكون مضغوط zip، فكه وحتلاقي الـ APK جاهز.

مقارنة سريعة

Codemagic: واجهة مرئية أسهل، مخصصة لتطبيقات الموبايل، بريد إلكتروني عند الانتهاء.
GitHub Actions: مجاني بالكامل (2000 دقيقة/شهر)، مدمج في GitHub، يحتاج خطوات أكثر بس أقوى في التخصيص.

12. ملف codemagic.yaml المرجعي

هاد هو الملف الجاهز اللي أضفناه لمشروع Breathy. تقدر تعدّله حسب حاجتك:

workflows: android-debug: name: "Breathy Debug APK" max_build_duration: 60 instance_type: linux_x2 environment: vars: PACKAGE_NAME: "com.breathy" GOOGLE_SERVICES_JSON: "" scripts: - name: "Set up local properties" script: | echo "sdk.dir=$ANDROID_SDK_ROOT" > local.properties - name: "Restore google-services.json" script: | if [ -n "$GOOGLE_SERVICES_JSON" ]; then echo "$GOOGLE_SERVICES_JSON" > app/google-services.json fi - name: "Get Gradle wrapper" script: | chmod +x gradlew || true - name: "Build Debug APK" script: | ./gradlew assembleDebug artifacts: - app/build/outputs/apk/debug/*.apk publishing: email: recipients: - your-email@example.com notify: success: true failure: true
تخصيص البريد الإلكتروني

غيّر your-email@example.com إلى بريدك الإلكتروني الفعلي عشان توصلك إشعارات البناء. عدّل الملف في GitHub وCodemagic يقرأ النسخة الجديدة تلقائياً.

ملخص الخطوات
